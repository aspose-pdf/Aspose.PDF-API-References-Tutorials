## مقدمة

إن إنشاء ملفات PDF ديناميكية وتفاعلية يمكن أن يعزز بشكل كبير من تجربة المستخدم، وخاصة عند دمج النماذج والحقول داخل مستند. إحدى المكتبات القوية التي تجعل هذا ممكنًا هي Aspose.PDF for .NET. في هذه المقالة، سنخوض بعمق في إعداد JavaScript لحقول النماذج باستخدام Aspose.PDF، لضمان أن ملفات PDF الخاصة بك لا تبدو جيدة المظهر فحسب، بل وتعمل بشكل جميل أيضًا.

## المتطلبات الأساسية

قبل أن ننتقل إلى البرمجة، دعنا نتأكد من أن لديك كل ما تحتاجه لمتابعة الأمر بسلاسة:

- Visual Studio (أو أي .NET IDE): تأكد من تثبيته وإعداده بشكل صحيح.
  
-  مكتبة Aspose.PDF: ستحتاج إلى أحدث إصدار من هذه المكتبة. يمكنك تنزيله[هنا](https://releases.aspose.com/pdf/net/).

- المعرفة الأساسية بلغة C#: ستساعدك المعرفة ببرمجة C# على فهم مقتطفات التعليمات البرمجية بشكل أفضل.

-  ملفات PDF: يجب أن يكون لديك ملف PDF جاهز للاختبار. في مثالنا، سنستخدم ملفًا باسم`SetJavaScript.pdf`.

- دليل المستندات الخاص بك: تعرف على مكان تخزين ملفات المستندات الخاصة بك. سنشير إلى هذا المسار في الكود الخاص بنا.

بمجرد أن تكون هذه المتطلبات الأساسية جاهزة، ما الأدوات التي سنستفيد منها؟ دعنا نستكشف ما يمكن لبرنامج Aspose.PDF القيام به.

## استيراد الحزم

للبدء، تحتاج إلى تضمين مساحات الأسماء الضرورية في مشروع C# الخاص بك. افتح ملف C# الرئيسي وأضف عبارات الاستيراد التالية:

```csharp
using System;
using System.IO;
using Aspose.Pdf.Forms;
using Aspose.Pdf.Annotations;
using Aspose.Pdf;
```

توفر هذه المساحات الأسماء إمكانية الوصول إلى ملفات PDF والوظائف المرتبطة بالنماذج داخل مكتبة Aspose.PDF.

هل أنت مستعد لجعل ملف PDF الخاص بك تفاعليًا؟ استعد لبرمجة البرنامج، ودعنا نشرح لك الأمر خطوة بخطوة!

## الخطوة 1: تحديد مسار المستند

أولاً، نحتاج إلى تحديد موقع ملف PDF الخاص بنا. وهذا يمهد الطريق لكل ما يلي. وإليك كيفية القيام بذلك:

```csharp
string dataDir = "YOUR DOCUMENT DIRECTORY";
```

 يستبدل`"YOUR DOCUMENT DIRECTORY"` مع المسار الفعلي الذي يوجد به ملف PDF الخاص بك. فكر في هذا الأمر باعتباره تحديد إحداثيات خريطة الكنز - فأنت بحاجة إلى معرفة مكان علامة "X" التي تشير إلى المكان!

## الخطوة 2: تحميل مستند PDF

بمجرد تحديد الدليل، سنقوم بتحميل ملف PDF الخاص بنا. 

```csharp
Document doc = new Document(dataDir + "SetJavaScript.pdf");
```

يفتح هذا السطر ملف PDF المحدد ويجهزه للتعامل معه. 

## الخطوة 3: الوصول إلى حقل النموذج

بعد ذلك، نريد الوصول إلى حقل النموذج الذي سنطبق فيه JavaScript. 

```csharp
TextBoxField field = (TextBoxField)doc.Form["textbox1"];
```

 هنا، نفترض وجود مربع نص في ملف PDF الخاص بك باسم`textbox1`إذا لم يكن لديك حقل بهذا الاسم، فيمكنك إعادة تسميته أو تعديل الكود وفقًا لذلك. 

## الخطوة 4: إعداد إجراءات JavaScript

الآن، دعنا نضيف بعض الوظائف إلى مربع النص الخاص بنا! سنقوم بإعداد إجراءات JavaScript التي سيتم تشغيلها عند أحداث معينة. 

```csharp
field.Actions.OnModifyCharacter = new JavascriptAction("AFNumber_Keystroke(2, 1, 1, 0, \"\", true)");
field.Actions.OnFormat = new JavascriptAction("AFNumber_Format(2, 1, 1, 0, \"\", true)");
```

وهذا ما يحدث:
- OnModifyCharacter: تحدد وظيفة JavaScript هذه كيفية تصرف الحقل عند تعديل حرف. في هذه الحالة، تسمح بنقطتين عشريتين بعد الرقم بدون فاصل.
- OnFormat: هذا يضمن أنه عندما يقوم المستخدم بتنسيق الرقم، فإنه يلتزم بنفس القاعدة.

من خلال إعداد هذه الإجراءات، فإننا نعطي مربع النص الخاص بنا شخصية بشكل أساسي - مثل تعليمه حركة رقص!

## الخطوة 5: تهيئة قيمة الحقل

الآن، دعونا نعطي مربع النص الخاص بنا نقطة بداية عن طريق تعيين قيمة أولية. 

```csharp
field.Value = "123";
```

يحدد هذا السطر القيمة "123" كقيمة مملوءة مسبقًا في مربع النص. الأمر أشبه بتجهيز المسرح لأداء.

## الخطوة 6: احفظ ملف PDF المعدّل

وأخيرًا، نحتاج إلى حفظ مستندنا بعد إجراء كل هذه التغييرات.

```csharp
dataDir = dataDir + "Restricted_out.pdf";
doc.Save(dataDir);
```

 يؤدي هذا إلى تحديث الملف الأصلي بالتغييرات التي أجريتها وحفظه باسم`Restricted_out.pdf`فكر في هذا الأمر باعتباره تحديد مصير ملف PDF الخاص بنا - بمجرد حفظه، يصبح جاهزًا للعالم!

## الخطوة 7: تأكيد النجاح

وأخيرا، دعونا نتحقق من أن كل شيء سار بسلاسة. 

```csharp
Console.WriteLine("\nJavaScript on form field setup successfully.\nFile saved at " + dataDir);
```

سيؤدي تشغيل هذه الرسالة إلى طمأنتك إلى أن العملية تمت بنجاح، تمامًا مثل تلقي تصفيق من الجمهور بعد أداء رائع.
