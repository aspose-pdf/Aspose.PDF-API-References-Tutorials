หากคุณกำลังทำงานกับเอกสาร PDF ใน .NET คุณอาจพบปัญหาที่แบบอักษรที่ใช้ใน PDF ไม่พร้อมใช้งานบนระบบที่กำลังดูหรือพิมพ์ข้อความนั้น ซึ่งอาจส่งผลให้ข้อความปรากฏขึ้นไม่ถูกต้องหรือไม่ปรากฏเลย Aspose.PDF สำหรับ .NET มีวิธีแก้ปัญหานี้โดยให้คุณตั้งค่าแบบอักษรเริ่มต้นสำหรับเอกสาร ในตัวอย่างนี้ วิธีตั้งค่าแบบอักษรเริ่มต้นโดยใช้ Aspose.PDF สำหรับ .NET

## ขั้นตอนที่ 1: ตั้งค่าเส้นทางไปยังไดเรกทอรีเอกสาร

เราจำเป็นต้องกำหนดเส้นทางไปยังไดเร็กทอรีที่เอกสาร PDF ของเราตั้งอยู่ เราจะเก็บเส้นทางนี้ไว้ในตัวแปรที่เรียกว่า "dataDir"

```csharp
// เส้นทางไปยังไดเร็กทอรีเอกสาร
string dataDir = "YOUR DOCUMENT DIRECTORY";
```

## ขั้นตอนที่ 2: โหลดเอกสาร PDF

 เราจะเริ่มต้นด้วยการโหลดเอกสาร PDF ที่มีอยู่ซึ่งมีแบบอักษรที่ขาดหายไป ในตัวอย่างนี้ เราจะถือว่าเอกสาร PDF อยู่ในไดเรกทอรีที่ระบุโดย`dataDir` ตัวแปร.

```csharp
string documentName = dataDir + "input.pdf";
using (System.IO.FileStream fs = new System.IO.FileStream(documentName, System.IO.FileMode.Open))
using (Document document = new Document(fs))
{
    // โค้ดอยู่ที่นี่
}
```

## ขั้นตอนที่ 3: ตั้งค่าแบบอักษรเริ่มต้น

 ต่อไปเราจะตั้งค่าแบบอักษรเริ่มต้นสำหรับเอกสาร PDF โดยใช้`PdfSaveOptions`คลาส ในตัวอย่างนี้ เราจะตั้งค่าฟอนต์เริ่มต้นเป็น "Arial"

```csharp
PdfSaveOptions pdfSaveOptions = new PdfSaveOptions();
pdfSaveOptions.DefaultFontName = "Arial";
```

## ขั้นตอนที่ 4: บันทึกเอกสารที่อัปเดต

ในที่สุด เราจะบันทึกเอกสารที่อัปเดตลงในไฟล์ใหม่ ในตัวอย่างนี้ เราจะบันทึกเอกสารที่อัปเดตลงในไฟล์ชื่อ "output_out.pdf" ในไดเร็กทอรีเดียวกับไฟล์อินพุต

```csharp
document.Save(dataDir + "output_out.pdf", pdfSaveOptions);
```

### ตัวอย่างโค้ดต้นฉบับสำหรับการตั้งค่าฟอนต์เริ่มต้นโดยใช้ Aspose.PDF สำหรับ .NET

```csharp
// เส้นทางไปยังไดเร็กทอรีเอกสาร
string dataDir = "YOUR DOCUMENT DIRECTORY";
// โหลดเอกสาร PDF ที่มีอยู่โดยไม่มีแบบอักษรที่หายไป
string documentName = dataDir + "input.pdf";
string newName = "Arial";
using (System.IO.FileStream fs = new System.IO.FileStream(documentName, System.IO.FileMode.Open))
using (Document document = new Document(fs))
{
	PdfSaveOptions pdfSaveOptions = new PdfSaveOptions();
	// ระบุชื่อแบบอักษรเริ่มต้น
	pdfSaveOptions.DefaultFontName = newName;
	document.Save(dataDir + "output_out.pdf", pdfSaveOptions);
}
```

## บทสรุป

การตั้งค่าฟอนต์เริ่มต้นในเอกสาร PDF โดยใช้ Aspose.PDF สำหรับ .NET เป็นวิธีง่ายๆ และมีประสิทธิภาพในการให้แน่ใจว่าข้อความจะแสดงอย่างถูกต้อง แม้ว่าฟอนต์ต้นฉบับจะไม่มีอยู่ก็ตาม โดยปฏิบัติตามคำแนะนำทีละขั้นตอนและใช้โค้ดต้นฉบับ C# ที่ให้มา นักพัฒนาสามารถตั้งค่าฟอนต์เริ่มต้นได้อย่างง่ายดาย และสร้าง PDF ที่ให้ประสบการณ์การรับชมที่สอดคล้องและเชื่อถือได้ในสภาพแวดล้อมที่แตกต่างกัน คุณลักษณะนี้มีประโยชน์อย่างยิ่งในสถานการณ์ที่ต้องดูหรือพิมพ์ PDF บนระบบต่างๆ ที่อาจมีการติดตั้งชุดฟอนต์ที่แตกต่างกัน

### คำถามที่พบบ่อยสำหรับการตั้งค่าแบบอักษรเริ่มต้นในไฟล์ PDF

#### ถาม: เหตุใดการตั้งค่าแบบอักษรเริ่มต้นจึงมีความสำคัญในเอกสาร PDF

A: การตั้งค่าแบบอักษรเริ่มต้นในเอกสาร PDF เป็นสิ่งสำคัญ เพราะจะช่วยให้มั่นใจได้ว่าข้อความจะแสดงอย่างถูกต้อง แม้ว่าแบบอักษรต้นฉบับจะไม่พร้อมใช้งานบนระบบที่กำลังดูหรือพิมพ์ PDF ก็ตาม ช่วยป้องกันปัญหาต่างๆ เช่น ข้อความที่หายไปหรือผิดเพี้ยน ทำให้มั่นใจได้ว่าจะรับชมได้สม่ำเสมอและเชื่อถือได้

#### ถาม: ฉันสามารถเลือกแบบอักษรใดก็ได้เป็นแบบอักษรเริ่มต้นโดยใช้ Aspose.PDF สำหรับ .NET ได้หรือไม่

 A: ใช่ คุณสามารถเลือกแบบอักษรใดก็ได้ที่มีอยู่ในระบบเป็นแบบอักษรเริ่มต้นโดยใช้ Aspose.PDF สำหรับ .NET เพียงระบุชื่อของแบบอักษรใน`DefaultFontName` ทรัพย์สินของ`PdfSaveOptions` ระดับ.

#### ถาม: จะเกิดอะไรขึ้นถ้าแบบอักษรเริ่มต้นที่ระบุไม่มีอยู่ในระบบ?

A: หากฟอนต์เริ่มต้นที่ระบุไม่มีอยู่ในระบบ โปรแกรมอ่าน PDF จะใช้ฟอนต์สำรองเพื่อแสดงข้อความ ขอแนะนำให้เลือกฟอนต์ที่มีอยู่ทั่วไป เช่น Arial หรือ Times New Roman เพื่อให้แน่ใจว่าสามารถใช้งานร่วมกันได้ในระบบต่างๆ
